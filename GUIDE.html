Overview

Excel Workbook Manager is a standalone Windows desktop application that sits alongside Excel and automates repetitive workbook management tasks — particularly those that arise during period-end actuarial processes such as IFRS 17 reporting.

Rather than manually hunting through Excel menus, it provides a unified control panel to close workbooks, relink external references, refresh calculation tabs, scan for errors, and much more — all without touching a macro or VBA editor.

No installation required. The tool ships as a single .exe file. Copy it to any Windows machine that has Excel installed and double-click to run.

Requirements

  • Windows 10 or 11
  • Microsoft Excel (desktop, any recent version)
  • Excel must be open and running before launching the tool
  • No Python installation needed on the target machine
Important: Always open Excel first, then launch the tool. If Excel is not running you will see a startup error.

Quick Start

  1. Open Excel Load the workbooks you want to work with — the tool reads whatever is currently open.
  2. Launch ExcelWorkbookManager.exe A splash screen appears briefly; the main window opens automatically.
  3. Click "Refresh All Lists" The bottom-left button re-scans all open workbooks. Use it any time you open or close a file in Excel.
  4. Select a tab and work Each tab is self-contained. Hover over ℹ buttons for inline explanations.

Smart Relinker — How It Works

The Smart Relinker is designed around the IFRS 17 folder convention where both the year folder and the period token in file names change every quarter:

I:\IFRS 17\2025\202512\Working\Non-model\RI Hub\RI Hub AoM 25FY.xlsx ↓ Period Shift: 202512 → 202603 + Rename rule: 25FY → 26Q1 I:\IFRS 17\2026\202603\Working\Non-model\RI Hub\RI Hub AoM 26Q1.xlsx

Pick one of four modes at the top of the tab — the UI below the radio row changes automatically to show only the controls that apply. In every mode the resulting Before / After paths land in the same Preview grid, which is fully editable before Apply Now.

① Period Shift mode

Enter an Old Period and a New Period (6-digit YYYYMM). The tool automatically replaces:

  • The year folder — e.g. \2025\ → \2026\
  • Every occurrence of the period token — e.g. 202512 → 202603

File name suffixes like 25FY or FY are not changed by the period shift alone — add them as Extra Rename Rules in the rules list below the mode selector, or simply load a saved profile.

② Custom Find & Replace mode

Define a list of Find / Replace pairs applied in sequence to every selected link path. Useful for one-off corrections, non-standard folder structures, or pure file-name suffix changes with no period shift. The rule list supports re-ordering, enabling / disabling individual rules, and loading from a profile.

③ Folder Match (fuzzy scan) — NEW

When to use it. The folder layout or file names on the new drive have changed in ways you cannot predict with a simple Find / Replace — maybe the vendor renamed files, the subfolder tree was restructured, or the period token moved from the end of the name to the middle. Point the tool at the new root and it figures out which file matches which old link.

Select Folder Match as the mode, set the new root folder, and click Scan & Match. The tool walks the folder recursively, indexes every .xlsx / .xlsb / .xlsm / .xls file it finds, and for each old linked file computes a fuzzy similarity score against every candidate.

How the fuzzy match works

Each filename is reduced to a skeleton before comparison, so volatile tokens do not artificially lower the score:

RI Hub AoM 25FY 20251231 v1.2.xlsx ↓ strip 8-digit date (20251231) ↓ strip 6-digit period token (no match here) ↓ strip version string (v1.2) ↓ lower-case, collapse spaces / underscores "ri hub aom 25fy" ← skeleton used for the similarity score

Skeletons are compared with Python's difflib.SequenceMatcher, producing a score from 0.00 to 1.00. Candidates below the threshold (default 70%) are discarded. Files sitting in the same relative subfolder as the original link are boosted to the top of the candidate list — cross-extension matches (e.g. .xlsm → .xlsx) are penalised by 5%.

What happens for each old link

  • One strong match — auto-populated in the preview with method tag Folder Match (92%).
  • Multiple close matches — a picker dialog appears showing every candidate with its score, version string, whether it lives in the same subfolder, and full path. Double-click a row or click Use Selected to accept it (method tag Folder Match (picked)). Click Skip to leave that row alone.
  • No match above threshold — the row is kept unchanged and tagged No Match so you can fix it manually.

Good to know

  • Folder Match does not require an Old Period / New Period — you can use it on its own, or combine it with a subsequent Find & Replace pass on the preview grid for last-mile corrections.
  • The scan is purely filesystem-based — no Excel calls, no COM, no workbooks opened. Large trees (10,000+ files) are typically scanned in a couple of seconds thanks to a length-based early-out filter that skips candidates whose maximum possible score is already below the threshold.
  • Excel must still be open when you click Apply Now, because that step rewrites each link via COM on the live workbook.

④ Manual Edit mode

Loads all selected links into the preview exactly as they are — no rules applied. Select any row, type a corrected path in the Manual Override box, click Update Row, then Apply Now when you are done. Use this when the changes are too varied or ad-hoc to describe with rules or fuzzy matching.

Preview grid — shared by all four modes

The preview grid is the single "commit surface" for every relink. Each row shows the workbook, the old path, the computed new path, the method that produced it (Period Shift / Find & Replace / Folder Match / Manual), and a ✔ / ✘ status from the Verify new paths exist check.

  • Manual Override — overwrite the new path on any individual row regardless of which mode produced it.
  • Open in Excel — send the full Before / After table into a live spreadsheet, edit the yellow New Path column directly in Excel cells, then click Import from Excel to pull the edits back into the preview.
  • Col width — widen the path columns with the spinner in the toolbar, or drag the column header dividers.
  • Verify new paths exist — keep this checkbox on. If any target file is missing on disk, you are warned before any link is rewritten.

Period Rollover Wizard

Use this tab when you need to roll multiple workbooks at once from one reporting period to the next. It combines relinking, rename rules, and tab refresh in a single guided flow.

  1. Select workbooks All open workbooks are listed and pre-selected. Deselect any you want to skip.
  2. Set periods or load a profile Enter Old Period and New Period, or pick a saved profile from the dropdown (which pre-fills both periods and all rename rules automatically).
  3. Add extra rename rules if needed e.g. 25FY → 26Q1. These are already included if you loaded a standard quarterly profile.
  4. Preview All Changes Review the full list of Before / After paths across every selected workbook.
  5. Apply Rollover Applies all relinks. If Run Tab Refresh after relink is checked, every sheet in every affected workbook is recalculated automatically.

Config Profiles

Profiles save a complete set of relinking rules (period shift + rename rules) so you never need to re-enter them each quarter. They are stored in profiles.json alongside the .exe file.

Q1 2026 → HY 2026 202603 → 202606 26Q1→26HY  ·   Q1 → HY
HY 2026 → Q3 2026 202606 → 202609 26HY→26Q3  ·   HY → Q3
Q3 2026 → FY 2026 202609 → 202612 26Q3→26FY  ·   Q3 → FY

Adding your own profiles

Open profiles.json in Notepad or any text editor. Each profile follows this structure:

{ "name": "My Custom Roll", "description": "Optional note about this profile", "period_shift": { "old": "202512", "new": "202603" }, "rules": [ { "find": "25FY", "replace": "26Q1" }, { "find": " FY", "replace": " Q1" } ] }
Tip: You can have a profile with only rules and no period_shift block — useful for pure find-and-replace scenarios.

Workbook Dependency Map

Click Build Dependency Map to scan all open workbooks and produce an expandable tree:

  • Green entries — the linked file is also currently open in Excel
  • Grey entries — the linked file is external (not open)
  • The # Links column shows how many cells in the source workbook point to that target

Use Export to Excel to save the full map as a formatted spreadsheet — handy for documentation or reviewing the refresh order before a period-end run.

Excel Instance Picker

If you have more than one Excel.exe running simultaneously, the Excel: dropdown in the bottom bar lets you control which instance the tool talks to.

  • [ALL INSTANCES] — the default. Workbook lists aggregate every open file across all instances. When a workbook name exists in only one instance it is shown as-is; when the same name appears in multiple instances the label includes a [PID N] suffix so operations are routed unambiguously.
  • PID XXXX — Book.xlsx — selecting a specific entry locks all operations (open, close, relink, calculate…) to that Excel process. Useful when you want to relink files in one model without accidentally touching a separate instance.
  • Click Rescan next to the dropdown to repopulate the list after launching or closing an Excel instance.
  • Your selection is saved between sessions. If the saved PID is gone when the tool next starts, it automatically falls back to [ALL INSTANCES].

Tips & Notes

  • Dark / Light mode — click the 🌙 Dark Mode button in the bottom bar to toggle between themes instantly.
  • Column widths — in the Smart Relinker preview, use the Col width spinner to widen path columns, or drag the column header dividers directly.
  • Preview before applying — always use Preview Changes before clicking Apply Now. The preview table is fully editable; individual rows can be overridden using the Manual override box.
  • Verify paths — keep the Verify new paths exist checkbox on. If a target file is not found on disk, the tool warns you before writing any changes.
  • Refresh All Lists — click this whenever you open or close workbooks in Excel so the tool's dropdowns stay in sync. (It also runs automatically every 20 seconds in the background.)
  • Folder Match threshold — the fuzzy matcher defaults to 70% similarity. If you are getting too many No Match rows on a reorganised folder tree, lower it via the Min score spinner; if too many wrong rows auto-populate, raise it toward 80–90% so only very confident matches apply and everything else falls into the candidate picker.
  • Queue All Modified workflow — edit the preview grid for one workbook (via any mode), click Queue All Modified, then switch to the next workbook and repeat. When all workbooks are prepared, click Apply Queue once to commit everything in sequence.
  • Open Current Tab’s Links only detects links written directly as 'path\[Book.xlsx]Sheet'!cell formula references. Links that come from defined names, Power Query, or DDE connections are not included — use List Links + Open Selected for those.
  • profiles.json — edit this file in Notepad to customise quarterly rollover rules. Changes take effect next time you click Refresh All Lists or open the relinker.
